# Dice Hero · 分辨率与屏幕适配规范

> 本文是项目唯一的分辨率与适配规范，任何场景搭建、UI 节点放置、文字字号设定都必须遵守。
> 适用引擎：Cocos Creator 3.8.7+
> 目标平台：微信小游戏（首发）/ 抖音小游戏（后续）/ Web

---

## 一、设计分辨率（硬性规范）

| 项                 | 值              |
| ------------------ | --------------- |
| 设计宽度           | **720 px**      |
| 设计高度           | **1280 px**     |
| 屏幕方向           | **竖屏（Portrait）** |
| 比例               | 9:16            |
| 像素风资产倍率     | 1× （不缩放）   |

### 在 Cocos Creator 中的设置位置

```
项目 → 项目设置 → 项目数据 → 设计分辨率
  Width:  720
  Height: 1280

Canvas 节点 → cc.Canvas 组件
  Design Resolution: 720 × 1280
  Fit Width:  ✅ 勾选
  Fit Height: ❌ 不勾选
```

---

## 二、屏幕适配策略（Fit Width）

**唯一允许的策略：Fit Width**

| 策略         | 行为                                         | 用否 |
| ------------ | -------------------------------------------- | ---- |
| Fit Width    | 宽度撑满屏幕，高度按设备比例延伸             | ✅ 用 |
| Fit Height   | 高度撑满屏幕，左右出现黑边/裁切              | ❌ 不用 |
| Fixed Width  | 等同 Fit Width 但只锁宽度                    | 等价 |
| Fixed Height | 同上                                          | ❌    |
| Show All     | 完整显示，可能上下/左右黑边                  | ❌    |
| Exact Fit    | 强行拉伸，会变形                             | ❌    |

### 为什么选 Fit Width

- 移动设备宽高比五花八门（19.5:9 / 20:9 / 21:9 / iPad 4:3 …）
- Fit Width 让 720 宽度始终铺满，**高度按设备比例自动延伸**
- 实际可视高度因设备而异：
  - 19.5:9 手机 → 实际 720×1560
  - 20:9 手机 → 实际 720×1600
  - iPad 4:3 → 实际 720×960（这种情况下游戏会"变矮"）

## 三、安全区与可视区

### 区域划分（基于 720×1280 设计稿）

```
┌──────────────────────────┐  y = +640    上边缘
│  顶部装饰区  ↕ 100px      │  y = +540
├──────────────────────────┤  y = +540    顶部安全线
│                          │
│                          │
│                          │
│      核心交互区          │  720 × 1080
│   （所有重要 UI 元素）    │  (-360 ~ +360, -480 ~ +540)
│                          │
│                          │
│                          │
├──────────────────────────┤  y = -480    底部安全线
│  底部装饰区  ↕ 60px       │  y = -540
└──────────────────────────┘  y = -640    下边缘
```

### 节点坐标系

Cocos Creator 默认坐标系：**原点在 Canvas 中心，Y 轴向上为正**。

| 位置           | x 范围        | y 范围        |
| -------------- | ------------- | ------------- |
| 全屏可视区     | -360 ~ +360   | -640 ~ +640   |
| 顶部装饰区     | -360 ~ +360   | +540 ~ +640   |
| **核心交互区** | **-360 ~ +360** | **-480 ~ +540** |
| 底部装饰区     | -360 ~ +360   | -640 ~ -540   |

### 安全区硬性规则

1. **关键交互按钮**（确认、取消、技能键、骰子）必须在核心交互区内
2. **血条、状态栏、章节标题** 可放顶部装饰区，但不能挡住关键信息
3. **Home Indicator 避让**：底部 60px 内不放任何可点击元素
4. **刘海/状态栏避让**：顶部 100px 内不放关键文本

## 四、UI 元素尺寸规范

### 字号（基于 720 宽度）

| 用途                | 字号    | 中文清晰度阈值 |
| ------------------- | ------- | -------------- |
| 主标题              | 56 px   | ✅ 大屏可见    |
| 副标题 / 章节名     | 44 px   | ✅              |
| 按钮文字            | 36 px   | ✅              |
| 正文 / 数值         | 28 px   | ✅              |
| 小字 / 提示         | 24 px   | ⚠️ 下限         |
| 禁用：              | < 24 px | ❌ 视觉过小    |

### 按钮尺寸

| 类型                | 最小高度 | 推荐高度 |
| ------------------- | -------- | -------- |
| 主操作按钮（确认）  | 80 px    | 100 px   |
| 次操作按钮          | 60 px    | 80 px    |
| 图标按钮（设置等）  | 60×60    | 80×80    |
| 骰子卡牌            | 120×160  | 140×180  |

### 间距

| 用途             | 像素    |
| ---------------- | ------- |
| 元素紧密排列     | 8 px    |
| 同组元素间距     | 16 px   |
| 不同组元素间距   | 32 px   |
| 区块分隔         | 48 px   |
| 页面级安全间距   | 60 px   |

## 五、像素风专项规范

### 渲染设置

```
项目 → 项目设置 → 项目数据 → 默认纹理过滤 = Point（最近邻采样）
```

- 像素风**严禁**使用 Linear / Trilinear 过滤
- Sprite 节点的 `Trim` 功能必须保留默认（True），裁切透明像素

### 像素对齐

- 所有像素图标的 **x、y 坐标必须为整数**
- Anchor Point 推荐 `(0.5, 0.5)`，避免亚像素错位
- 节点 Scale 只允许 `1, 2, 3, 4` 等整数倍，禁用 1.5、2.5

### 像素图标尺寸建议

- 小图标：16×16, 24×24, 32×32（不使用 20×20 等非 8 倍数）
- 中图标：48×48, 64×64
- 大图标：96×96, 128×128
- 角色立绘：256×256, 512×512

## 六、检查清单（场景搭建后必跑）

完成一个场景后，按下面 checklist 验收：

- [ ] Canvas 节点设计分辨率 720×1280
- [ ] Canvas 节点 Fit Width 勾选，Fit Height 未勾选
- [ ] 所有关键按钮 y 坐标在 -480 ~ +540 之间
- [ ] 所有点击元素最小尺寸 ≥ 60×60
- [ ] 所有文字字号 ≥ 24 px
- [ ] 像素风纹理过滤为 Point
- [ ] 节点坐标全部为整数（无小数点）
- [ ] 不同长宽比设备模拟测试（19.5:9 / 20:9 / iPad）
- [ ] 截图全屏显示，无变形、无黑边、无关键信息被裁

## 七、常见错误与对应处理

| 错误现象                       | 原因                          | 修正                              |
| ------------------------------ | ----------------------------- | --------------------------------- |
| iPhone X 上按钮被刘海挡住      | 顶部安全线没遵守              | 关键按钮 y < +540                  |
| iPad 上画面"变矮"              | Fit Width 在 4:3 设备的固有现象 | 顶部/底部装饰区放纯背景，不放关键信息 |
| 像素图变模糊                   | 纹理过滤是 Linear              | 改为 Point                        |
| 文字在小屏看不清               | 字号 < 24 px                   | 至少 24 px，重要文字 28 px 起     |
| 按钮点不准                     | 按钮尺寸 < 60×60               | 强制 ≥ 60×60                       |
| 切换设备后 UI 错位             | 用了 Fit Height 或 Show All    | 改为 Fit Width                    |
| 像素图边缘有锯齿/不齐          | Anchor 不是 (0.5, 0.5) 或坐标有小数 | Anchor 0.5/0.5，坐标取整          |
